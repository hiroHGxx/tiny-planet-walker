# Tiny Planet Walker

小さな球形の惑星の上を、ただ気持ちよく歩き回るだけの静かな箱庭作品です。

宇宙空間に浮かぶ緑の小惑星の表面をキャラクターが歩き、丸い地平線の向こうから木や岩が少しずつ現れます。ゴール・スコア・敵・アイテムはありません。

村人に近づくと頭の上にひとことつぶやきが出ます(昼と夜で内容が変わります)。薬草に近づくとその種類が「薬草図鑑」(右上の📖ボタン)に記録され、見つけた種類は次に遊ぶときも残ります。風・小鳥・夜の虫の音・足音の環境音も流れます(右上の🔊ボタンでミュートできます)。

## 使用技術

- [Three.js](https://threejs.org/) — 3D描画
- TypeScript — 型付きのJavaScript
- Vite — 開発サーバーとビルド
- npm — パッケージ管理

物理エンジンや外部の3Dモデル・画像素材は使っていません。すべてThree.jsのプリミティブ(球・円柱・円錐など)で作られています。

## 初回セットアップ

Node.js(v20以上を推奨)とnpmが必要です。

```bash
npm install
```

## 開発サーバーの起動

```bash
npm run dev
```

表示されたURL(通常は http://localhost:5173/ )をブラウザで開きます。

## ビルド

```bash
npm run build
```

TypeScriptの型チェックのあと、`dist/` フォルダに本番用ファイルが出力されます。

## テスト

```bash
npm test
```

球面移動(半径の維持・極通過)、衝突の押し出し、道路と湖の重なりを数値で検証します(Vitest)。

## 対応環境

WebGL 2に対応したブラウザ(Chrome / Firefox / Safari の最新版)が必要です。WebGLが使えない環境では画面に案内が表示されます。タッチ端末では画面左下に移動用の仮想スティックが表示されます。

## 操作方法

| キー | 動作 |
| --- | --- |
| W / ↑ | 画面の奥へ移動 |
| S / ↓ | 画面の手前へ移動(振り返って走る) |
| A / ← | 画面の左へ移動 |
| D / → | 画面の右へ移動 |
| マウスドラッグ | カメラをプレイヤーの周りに回す |
| マウスホイール | ズームイン / ズームアウト |
| 仮想スティック(タッチ端末) | 画面左下のスティックで移動 |
| タッチドラッグ | カメラをプレイヤーの周りに回す |
| 右上の🔊ボタン | 環境音のミュート切り替え |
| 右上の📖ボタン | 薬草図鑑を開く / 閉じる |

移動は「カメラ相対式」です。WASDは画面上の行きたい方向を表し、キャラクターは移動方向へ自動で振り向きます。キーを離すと止まります。ジャンプやダッシュはありません。カメラの向きはキャラクターの向きから独立しているので、移動中にドラッグして眺めを変えても、画面が勝手に回ることはありません。

## 球面移動の実装方法

この作品の核心は「平らな地面」ではなく「球の表面」を歩くことです。`src/player.ts` に日本語コメント付きで実装しています。考え方は次のとおりです。

1. **上方向 = 球面法線**
   惑星の中心はワールド原点なので、プレイヤーの位置ベクトルを正規化するだけで「その地点の上方向(球面法線)」が得られます。重力はその逆向き、という扱いです。

2. **移動方向はカメラ基準(カメラ相対式)**
   カメラの視線方向を接平面へ投影したものを「画面の奥」、それと法線の外積を「画面の右」として、WASD入力を移動方向に変換します。キャラクターの振り向きは、ワールドのY軸ではなく「いまいる地点の法線」を軸としたQuaternion回転で行うため、惑星のどこにいても同じ操作感になります。

3. **接平面上の移動**
   移動方向は `f - n(f・n)` で法線成分を取り除き、球面に接する方向だけを残してから移動します。

4. **移動後に半径へ正規化**
   接線方向へ直進すると球からわずかに離れるので、毎フレーム `position.normalize().multiplyScalar(半径)` で「中心から一定距離」へ戻します。これで何時間歩いても浮いたり沈んだりしません。

5. **姿勢の補正は「最小回転」で**
   移動して法線が変わったら、「今の上方向 → 新しい上方向」への最小回転を `Quaternion.setFromUnitVectors` で求めて姿勢に掛けます。`lookAt` のようにワールドの上方向を基準にしないため、北極・南極に相当する場所を通っても向きが突然反転しません。毎フレームQuaternionを正規化して誤差の蓄積も防いでいます。

6. **フレームレート非依存・GC対策**
   移動量はすべて `deltaTime`(前フレームからの経過秒)に比例させています。また、毎フレームの計算にはモジュール内で使い回す一時的な `Vector3` / `Quaternion` を使い、大量のオブジェクト生成を避けています。

カメラ(`src/camera.ts`)も同じ考え方です。カメラは「視線方向」(接平面上の単位ベクトル)を自分で保持し、ドラッグの周回角(法線軸の回転)とホイールの距離、見下ろし角から目標位置を決めて、位置・注視点・上方向を指数減衰で滑らかに補間します。視線方向は**キャラクターの向きから独立**しており、プレイヤーが球面上を移動したぶんは「前の法線 → 今の法線」への最小回転で運びます(平行移動)。これがカメラ相対移動の基準になり、キャラが振り向いても画面は回りません。補間中でも惑星の中心からの距離に下限を設け、カメラが惑星内部に入らないようにしています。

## あえて実装していないもの

- ジャンプ・ダッシュ
- 物理エンジン
- 外部3Dモデル・テクスチャ画像(すべてコードで生成)
- BGM・音声ファイル(環境音はWeb Audioでコード合成)
- ゴール・スコア・敵・アイテム・クエストなどのゲーム要素(記録するだけの薬草図鑑はあります)
- 地形の高低差(球面移動の安定性を最優先)

すべての形・色・動き・光をコードだけで作った、小さな箱庭の惑星です。
