## 3. 問題点

### M-01: モバイル画面ではプレイヤーを移動できない

重要度: 中

`viewport`設定とPointer Eventsによりスマートフォンでも画面は表示されますが、移動入力はWASD・矢印キーだけです。320px幅で実行した場合もHUDはキーボード操作だけを案内し、画面上の移動UIはありません。

また、Canvasの`touch-action`は`auto`で、`pointerdown`と`pointermove`でも`preventDefault()`やPointer Captureを使用していません。タッチドラッグがブラウザのスクロール・ピンチ操作に奪われ、`pointercancel`になる可能性があります。

該当箇所:

- `src/main.ts:33-68`
- `src/camera.ts:62-90`
- `src/style.css:11-15`
- `index.html:5`

推奨:

- モバイルを対応対象にする場合は仮想スティック等の移動入力を追加する。
- Canvasへ`touch-action: none`を指定する。
- ドラッグ開始時に`setPointerCapture()`を使用する。
- モバイル非対応なら対応環境をREADMEと画面上で明記する。

### M-02: 羽用の設定変更が共有マテリアル全体へ波及する

重要度: 中

`toonMaterial()`は色と自己発光色をキーにマテリアルを共有します。その戻り値へ直接`side = THREE.DoubleSide`を設定しているため、羽以外の同色オブジェクトも両面描画へ変わります。

具体例:

- ハチ・トンボの羽が`PALETTE.cloud`を変更し、雲や羊などへ波及する。
- 蝶の羽が`PALETTE.petal`や`PALETTE.accentPurple`を変更し、花やキノコなどへ波及する。

見た目の裏面表示が意図せず変化し、描画負荷も増加します。初期化順に依存するため、将来の配置順変更で見た目が変わる危険もあります。

該当箇所:

- `src/palette.ts:107-124`
- `src/ambient.ts:80-85`
- `src/animals.ts:302-303`
- `src/animals.ts:362-363`

推奨:

- 羽専用マテリアルを新規作成する。
- またはマテリアルキャッシュのキーへ`side`等の設定を含める。
- 共有マテリアル取得後のプロパティ変更を禁止する。

### M-03: 道路の交差点や施設境界でレンガが過密・重複する

重要度: 中

実行画面の開始地点付近で、複数の道路弧の始点・終点に同じ3列のレンガが重なり、交差点が大きなレンガの塊に見えます。同一高さに近いタイルが重なるため、輪郭や陰影も複雑になっています。

既存の`debug/`画像では、畑の土パッチを道路が横切る状態と、湖の砂・水際へ道路が食い込む状態も記録されています。

湖迂回処理は湖の外側に経由点を1点置くだけです。`from → detour`と`detour → to`の各弧が湖のクリアランス円へ接する保証がなく、経由点の手前で再び湖側へ切り込む可能性があります。また、迂回後の弧を再検証していません。

該当箇所:

- `src/world.ts:127-182`
- `src/town.ts:51-120`

推奨:

- 交差点付近では重複するタイルを距離判定で除外する。
- 交差点専用の中央パーツを作る。
- 湖の迂回には接点を2点使用するか、複数サンプルで全弧のクリアランスを検証する。
- 道路、湖、畑、家の最終配置後に重なりを検査するデバッグ表示を用意する。

### M-04: 飛行する虫は非表示になると経路更新も停止する

重要度: 中

ハチとトンボの可視判定は、前回更新時の`root.position`を使います。遠いと判定された場合は`insect.update(time)`が呼ばれないため、位置が古いまま固定されます。

その結果:

- 本来は時間とともに飛行経路を進む虫が、非表示中は停止する。
- プレイヤーが古い停止位置へ近づいた瞬間、現在時刻の経路位置へジャンプする。
- プレイヤーが停止位置へ近づかなければ、実際の経路が近くへ戻ってきても再表示されない。

該当箇所:

- `src/animals.ts:323-343`
- `src/animals.ts:383-411`
- `src/animals.ts:479-487`

推奨:

- 可視判定前に、現在時刻における位置だけは計算する。
- または`Insect`へ現在位置を取得する軽量関数を追加し、その結果で可視判定する。

### M-05: WebGLを作成できない環境では説明なしに初期化が停止する

重要度: 中

WebGLが無効または利用不能なChrome環境で、`new THREE.WebGLRenderer()`が例外を投げ、アプリ初期化が停止することを再現しました。画面にはHUDと空のCanvasだけが残り、利用者向けの説明は表示されません。

該当箇所:

- `src/main.ts:10-15`

推奨:

- レンダラー作成を`try/catch`で囲む。
- WebGLが利用できない場合は、対応ブラウザ・GPU設定についての案内を画面へ表示する。
- `webglcontextlost`イベントも監視する。

### L-02: 衝突解決が1回の走査だけで、密集地点では再侵入が残る可能性がある

重要度: 低

コライダーを先頭から1回だけ処理しています。後ろのコライダーによる押し出しで、すでに処理済みのコライダーへ再び侵入しても再判定されません。家・小物・NPC・柵が密集する場所で、振動、引っ掛かり、わずかなめり込みが発生する可能性があります。

該当箇所:

- `src/collision.ts:42-71`

推奨:

- 最大2〜4回程度の反復解決を行う。
- 1回の押し出し量と残存侵入数をデバッグ表示できるようにする。

### L-03: 本番JavaScriptが単一チャンクで大きい

重要度: 低

確認済みビルド結果:

- JavaScript: 約624.36KB
- gzip: 約160.58KB

Three.jsとポストプロセスが主要因と考えられます。初回ロード時間と低速回線での待ち時間に影響します。

推奨:

- 使用しているThree.jsアドオンと初期ロード範囲を確認する。
- 必要に応じて遅延ロード・コード分割を検討する。
