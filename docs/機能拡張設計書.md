# 機能拡張設計書 — 「薬草の星」v2 モリモリ計画

2026-07-18 作成・同日大幅拡張。既存の [仕様書](./仕様書.md) / [設計書](./設計書.md) の上に、
物語・生活・収集・旅の機能群を一気に足すための設計。

## 0. 方針

**「あとで消しやすい」ことを最優先の設計原則にする。**
雰囲気に合わない機能は捨てる前提で全部作るので、
1機能 = 1ディレクトリ、登録は1行、消しても他が壊れない構造にする。

### 決定事項(2026-07-18 ユーザー確認済み)

| 論点 | 決定 |
| --- | --- |
| 操作 | ボタン操作OK。ただし「1ボタン(E / タップ)」だけに抑える |
| 世界観テキスト | オリジナルで書く(`content/` 集約で後から差し替え可能に) |
| おえんちゃんの役割 | **薬師**。素材を集め、薬を調合し、届ける人 |
| スケール | ~~星は1つ~~ → **変更:何かを規定量ためると「次の星」へ旅立てる**メタ目標を置く |
| 会話UI | ノベル風の下部窓+丸フレーム似顔絵(3Dモデルの生中継方式) |
| 調合の場 | **家の中に入る3D室内シーン**を作り、そこの調合台で行う |
| セーブ | ブラウザローカル(localStorage)。開発中のリセットは許容 |
| 配信 | **静的なGitHub Pagesのまま**(§6参照。サーバーは不要) |

### 変わらない作法(既存方針の継承)

- 外部アセットなし。形・色・動き・光・音・挿絵すべてコード生成
- スコア・敵・体力・死・お金なし。報酬は「会話・ものがたり・眺め・旅」
- 実行時演出は `Math.random()` のみ。配置用シード乱数は絶対に消費しない
- 色は `palette.ts` に集約。トゥーン+フラット法線の見た目を守る
- 操作は移動+1ボタンだけ。管理・世話・ノルマの遊びにしない

## 1. アーキテクチャ

### 1.1 Feature モジュール

```
src/
  features/
    feature.ts      … Feature / FeatureContext 型定義
    registry.ts     … FEATURES 配列(ここから1行消せば機能が消える)
    events.ts       … 型付きイベントバス
    save.ts         … 機能ごとの localStorage 読み書き
    title/ interact/ pouch/ …(機能ごとに1ディレクトリ)
  content/
    npcs.ts recipes.ts letters.ts
    quests/         … 章ごとに分割し dynamic import で遅延読み込み
      chapter1.ts chapter2.ts … errands.ts(お手伝いテンプレ)
```

```ts
export interface FeatureContext {
  scene: THREE.Scene;
  player: Player;
  camera: THREE.PerspectiveCamera;
  world: World;
  events: EventBus;
  sunElevation(): number;
}

export interface Feature {
  id: string; // セーブキーにも使う(tiny-planet-walker:<id>)
  setup(ctx: FeatureContext): void;
  update?(deltaTime: number, ctx: FeatureContext): void;
}
```

- `main.ts` への変更は2箇所だけ:起動時に全 `setup()`、ループで全 `update()`
- 機能のCSSは `features/<name>/style.css` を機能側で `import`(消すとCSSも消える)
- 機能同士は直接 import しない。共有はイベントバスとセーブ経由

### 1.2 イベントバス(機能間の疎結合)

```ts
interface GameEvents {
  'herb-discovered': { species: string };
  'item-picked': { item: string };            // 薬草・羊毛・水・木の実…
  'item-changed': { item: string; count: number };
  'quest-started': { questId: string };
  'quest-completed': { questId: string };
  'letter-found': { letterId: string };
  'weather-changed': { weather: 'sunny' | 'cloudy' | 'rain' | 'fog' };
  'day-passed': { day: number };
  'npc-talked': { npcId: string };
  'craft-done': { recipeId: string };
  'scene-changed': { scene: 'planet' | 'interior' };
  'starlight-changed': { count: number };     // 星あかり(メタ進行の通貨)
}
```

発火側は購読者がゼロでも困らない(=機能を消しても発火側は壊れない)。

### 1.3 シーン切り替え(星の上 ⇄ 家の中)

- `SceneDirector` を新設:`'planet' | 'interior'` のアクティブシーンを持ち、
  暗転フェード → update/描画対象の切替 → 明転、を担う
- 星side(既存の world 一式)と室内side(新設)は別 `THREE.Scene`。
  室内は平面の床+矩形の壁で、**球面移動ではなく平面移動**の簡易モードを
  Player に追加する(入力・アニメは共通、位置計算だけ差し替え)
- 室内にいる間、星side の update は止める(時間は「日数時計」だけ進める)

### 1.4 既存モジュールへの「小さな公開口」

| ファイル | 追加する口 | 理由 |
| --- | --- | --- |
| `audio.ts` | `playJingle(kind)` / `setMelodyMode(mode)` | AudioContext を共有するため |
| `journal.ts` | 発見時に `herb-discovered` を emit(1行) | 依頼・ジングル連携 |
| `world.ts` | `herbSightings` に個株の `mesh` 参照を追加 | 摘む演出(縮小→再生) |
| `npc.ts` | `id` 付与と `hold()`(話しかけ中は立ち止まる) | 会話中に歩き去らないため |
| `player.ts` | 平面移動モード(室内用) | シーン切り替え |

### 1.5 セーブ

- キーは `tiny-planet-walker:<featureId>`。バージョン番号を同梱し、
  合わなければ黙って初期化(リセットOK方針)
- **最終プレイ時刻(タイムスタンプ)を必ず保存** → 放置進行(F19)の基礎
- 星が複数になったら、星ごとの状態は `tiny-planet-walker:planet:<n>:<featureId>`

### 1.6 ゲーム内時間と実時間

- ゲーム内「日数」:太陽が昇った回数を数えて `day-passed` を発火
  (薬草の復活・栽培の成長・掲示板の張り替えの時計)
- 実時間:起動時に「前回からの経過実時間」を計算し、留守の間の進行に変換
  (§F19。例:実1時間 = ゲーム内1日、上限つき)

## 2. 機能カタログ

### Phase 1 — 玄関と基盤

**F16 title(タイトル画面)** ★新規
- 起動時に紙調のタイトル:作品名+「はじめる」+小さな星の回転
  (実際の惑星シーンを遠景でゆっくり回すだけ。専用アートは作らない)
- 「はじめる」クリックが **AudioContext の解錠を兼ねる**
  (現状の「最初の操作まで無音」問題がここで自然に解決する)
- つづきから/はじめから(セーブ全消し)の2択。ミュート状態も表示

**F1 interact(1ボタン基盤)**
- 世界に `Interactable`(方向・半径・ラベル・優先度・onUse)を登録する仕組み
- 最寄り1件だけに紙調の小プロンプト(「E 話す」「E 摘む」「E 入る」)
- キーボード=E、タッチ=右下の丸ボタン。全採取・会話・扉の共通土台

**F2 pouch(採取とポーチ)**
- 薬草の個株を「E 摘む」→ ポーチへ(株はぷるんと縮み、ゲーム内2日で再生)
- HUDに🧺、紙調パネルで所持数。上限・重さなし(管理ゲーにしない)

**F3 music-v2(音の拡張)**
- 夜のメロディ(短調・遅め)+昼夜クロスフェード
- ジングル:発見・依頼達成・調合完了・旅立ち(3〜5音)

### Phase 2 — 物語の背骨

**F4 named-npcs(名前つき村人)+ 増員** ★拡張
- **NPCを20体 → 30体に増員**(村への追加+街道・湖畔・丘に新配置)
- うち10人前後に名前・肩書き・性格(`content/npcs.ts`)
  例:パン屋のマルさん、ひつじ番のトトさん、湖の釣り好きロコじいさん…
- ふだんは名前を出さない。話しかけたときだけ会話窓に名前が出る

**F5 talk(話しかけ)**
- 画面下部にノベル風の紙調ウィンドウ(名前+セリフ、クリック/Eで送り)
- 左の丸フレームに**似顔絵**:画像ではなく、話し相手の3Dモデルを
  似顔絵用カメラ+viewport/scissor でその場レンダリング(生中継方式)。
  位置合わせが崩れる環境向けに Canvas 2D 似顔絵への差し替え口を用意
- 素材フォルダのPNG(CryptoNinja IP未確認、gitignore済み)は公開物に使わない
- 会話中はNPCが立ち止まりプレイヤーの方を向く(`npc.hold()`)
- 選択肢は「うける / またこんど」だけ

**F6 quests(依頼)— 目標100本** ★拡張
- 2階建てにして100本を現実的にする:
  1. **手書きの物語依頼 約20本**:名前つき村人のチェーン。
     `content/quests/chapterN.ts` に章ごと分割、dynamic import で遅延読み込み
  2. **掲示板のお手伝い(テンプレ生成)実質無限**:村の広場に依頼掲示板を建て、
     ゲーム内日ごとに数件を生成。「ひつじの毛を2つ」「湖の水を汲んで」
     「まるば草を3株」「〇〇さんに薬を届けて」などのテンプレ×対象×数量の組合せ
- 薬の依頼だけでなく**町のお手伝い**(羊毛・水汲み・木の実拾い・届け物)を混ぜる
- 進行中の依頼はHUD左下にひとこと表示。同時受注は3件まで(あわてさせない)
- 達成報酬:お礼のセリフ+**星あかり**(F20)+ときどき種(F12)や手紙(F11)

**F7 crafting+interior(調合と家の中)** ★拡張
- おえんちゃんの家のドアに「E 入る」→ **3D室内シーンへ切り替え**(§1.3)
- 室内は外観より少し広い一間:調合台・薬棚・寝床・窓(外の昼夜が映る)・
  ポスト(F19の受け取り場所)。すべてプリミティブ生成
- 調合台で「E 調合」→ 紙調のレシピ帳UI。材料が揃っていれば「つくる」
  → 湯気の粒子+ジングル。レシピは10種程度から始めて依頼で増える
- 薬は届け物依頼の納品物になる(薬師の仕事の中心ループ:
  **摘む → 調合する → 届ける → お礼と星あかり**)

### Phase 3 — 世界の呼吸

**F8 weather(天気)**
- 晴/くもり/雨/霧を5〜10分単位でゆるく遷移。雨=Points+合成雨音、霧=fog
- `weather-changed` 発火(つぶやき・掲示板の依頼文面にも反映できる)

**F9 npc-schedule(村人の生活)**
- 夜は家路につき、うろつき半径と歩速が下がる。朝は持ち場へ

**F10 animals-friend(動物のなつき+羊毛)** ★拡張
- 動物に「E なでる」→ ハート粒子。なつくとついてくる
- **ひつじは「E 毛を刈る」で羊毛が採れる**(ゲーム内3日で復活。
  刈った後は一回り小さい見た目に)→ お手伝い依頼の納品物

**F11 letters(手紙の断片)**
- 星のあちこちに光る紙片8枚。「前の薬師さん」の日記が少しずつ読める
- 全部読むと小さな結び。**次の星へ行く動機の物語的な伏線**もここに仕込む

**F18 nature+town(構造物・自然物・採取物の増築)** ★新規
- 建物:風車、パン屋の窯、桟橋、井戸、鐘つき塔、掲示板(F6)、気球の発着台(F20)
- 自然物:木の実のなる木、きのこの輪、花畑の拡張、小川(見た目のみ)
- 採取物の追加:**湖の水(桟橋・井戸で汲む)・木の実・きのこ・花・羊毛**
- 薬草も**7種 → 12種**へ(新しい群生地+図鑑・挿絵も拡張)
- すべて既存の配置ルール(道・湖・村との距離チェック)とInstancedMesh/
  マテリアル共有・距離カリングに乗せる

### Phase 4 — 大きな流れと余韻

**F12 gardening(種まき)**
- 家の横に畑3マス。依頼のお礼の種を「E うえる」→ ゲーム内2〜3日で育つ
- 水やり・枯れなし(世話ゲーにしない)

**F19 idle(るすばんの星)** ★新規 — 放置ゲー要素
- 起動時に前回からの**実時間**を計算し、留守の間の出来事に変換:
  - 薬草・羊毛・畑が育っている(上限あり。実1時間=ゲーム内1日、最大7日分)
  - **ポストに手紙やお礼が届いている**(「るすの間に◯◯さんが来たみたい」)
  - たまに小さな贈り物(種・木の実)
- サーバー不要:タイムスタンプ差分だけで成立する(§6)
- 通知は出さない。「開いたら育っていた」の静かな喜びに留める

**F20 planets(次の星へ)** ★新規 — メタ目標
- 報酬の軸は「**星あかり**」(仮名):依頼達成・図鑑コンプ・手紙全読で貯まる、
  小さな光の粒。HUDには常時出さず、家の中のランタン瓶に溜まっていくのが見える
- 規定量たまると**気球の発着台**(F18)が使えるようになり、次の星へ旅立てる
- 次の星=新しいシードで生成した星(木・岩・薬草配置・NPC外見が変わる)+
  星ごとの色テーマ(パレット差し替え:秋の星、雪の星、花の星…)
- 元の星にはいつでも帰れる(気球で行き来)。進行は星ごとにセーブ(§1.5)
- 実装は「createWorld のパラメータ化+シーン再構築」。F15 seed-world は
  この機能に吸収する(URLシードは開発用の隠し機能として残す)

**F13 journal-v2(図鑑の思い出化)**
- 発見場所(星の小さな展開図に点)と日時を図鑑に記録。星ごとにページが分かれる

**F14 photo(写真モード)**
- 📷でUIを隠して構図モード → クリックでPNG保存+合成シャッター音

### 将来の予約席(v2では作らない)

**F21 multiplayer(いっしょに歩く)** ★予約のみ
- 他の人と同じ星を歩き、姿とつぶやきが見える(戦いも競いもしない)
- 方式は2段階を想定:
  1. 試作:WebRTCのP2P(Trystero等)。サーバー不要でGitHub Pages静的のまま、
     URLを送った友だち数人と同じ星に入れる
  2. 本格化:Cloudflare Workers + Durable Objects(またはPartyKit)で
     星=部屋のWebSocket同期。ゲーム本体はGitHub Pagesのままでよい
- Vercelは常時接続に不向きなため採用しない(検討済み・2026-07-18)
- F20のシード付きの星がそのまま部屋IDになる。同期は位置・向き・つぶやきのみ
  (毎秒10〜15回・数十バイト/人)。補間・入退室・名前表示が新規実装になる
- v2の設計はこの機能を前提にしない。ただしFeature方式なので、
  将来 `features/multiplayer/` を1ディレクトリ足すだけで済む構造は保つ

## 4. 実装順と検証

Phase 1 → 2 → 3 → 4 の順。**各Phaseの終わりに必ず:**

1. `npm test`(ポーチ増減・依頼の状態遷移・テンプレ生成・レシピ判定・
   天気遷移・日数カウント・放置進行の時間計算はすべてvitestで書ける)
2. `.claude/skills/verify` のブラウザ検証(コンソールエラー0、半径25.000維持、
   **FPS計測**を検証項目に追加)
3. 機能単位のコミット(消す判断がしやすいように)

## 5. 機能を消すときの手順

1. `features/registry.ts` から該当行を消す
2. `src/features/<name>/` を削除
3. `content/` の該当データファイルを削除(依頼の参照があれば依頼側も調整)
4. 既存モジュールの「口」は残してよい(呼ばれないだけで無害)
5. localStorage の残骸は無害

## 6. 技術ノート:静的Pagesのままでよい理由

- **依頼100本のテキストは gzip後 約30KB**(画像1枚以下)。読み込みの重さの
  問題にはならない。章ごとの dynamic import 分割は「行儀」としてやるだけ
- 放置進行は「保存したタイムスタンプとの差分」で完結し、サーバー不要
- 動的なバックエンドが必要になるのは:他人との星の共有・ランキング・
  プッシュ通知、を作るときだけ。現計画には含まれない
- したがって**配信はGitHub Pagesのまま**。転送量は全部足しても200KB台の見込み

## 7. v3構想:四季の星めぐり(2026-07-18 ユーザー発案・設計のみ)

### 星の構成(5つ)

| 星 | 役割 | 景観 | 固有薬草(案) |
| --- | --- | --- | --- |
| 薬草の星 | チュートリアル&ホーム。現状のまま | 緑の草原 | 基本7種+こがね穂・すずふり草 |
| はるかぜの星(春) | ✨5 | 桜の木(桃色の葉玉)・花びらの舞・若草色 | さくら草・わかば草 |
| なぎさの星(夏) | ✨15 | 大きな海と砂浜・ヤシ風の木・強い日ざし・波音 | しおかぜ草・ひまわり草 |
| もみじの星(秋) | ✨30 | 紅葉(木の葉色を差し替え)・きのこ多め・実りの畑 | もみじ茸・くりの実 |
| こなゆきの星(冬) | ✨50 | 雪の白い草地・降る雪・静かな冬のメロディ | ゆきわり草・こおり花 |

- 現在の「こもれびの星」「しんじゅの星」は秋・冬の星に改装して引き継ぐ
- あかね草→秋、むらさき茸→冬、月しろ草→冬、と既存の固有薬草も季節へ再配置

### 技術設計

- `setPlanetTheme` を拡張:草地の色に加えて **木の葉色・空の色味・天気の偏り
  (夏=晴れ多め、冬=雪=雨パーティクルの白色版)・BGMモード** を星ごとに切替
- 海(夏):LAKES等のレイアウト定数を「星ごとの上書き表」にして、
  夏の星だけ超大型の湖+広い砂浜の帯を定義する(レイアウト回転と併用)
- 雪(冬):weatherの雨Pointsを白色・低速に切り替えて常時弱く降らせる
- 季節の演出:春=花びら、秋=落ち葉のPointsをweather機能の枠で追加
- content/planets.ts に季節・薬草・住人・テーマ設定を集約(1星=1エントリ)
