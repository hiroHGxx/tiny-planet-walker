# Tiny Planet Walker 仕様書

作成日: 2026-07-16(ソースコードからのリバースエンジニアリングにより作成)
対象バージョン: 0.1.0

## 1. 概要

小さな球形の惑星「薬草の星」の上を、おえんちゃん(薬屋の女の子)として歩き回る箱庭作品。
ゴール・スコア・敵・アイテムはない。村人や動物が暮らす星を眺めて散歩することが目的。

- すべての形・色・動き・光をコードのみで生成(外部3Dモデル・画像テクスチャ不使用)
- 見た目の方向性: 積み木・ブリキのおもちゃ・絵本・ローポリ
- 使用技術: Three.js / TypeScript / Vite / Vitest

## 2. 対応環境

- WebGL 2対応ブラウザ(Chrome / Firefox / Safari の最新版)
- WebGLが使えない場合は画面中央に日本語の案内を表示して初期化を中断する
- タッチ端末(`pointer: coarse`)では画面左下に移動用の仮想スティックを表示する

## 3. 操作仕様

移動は**カメラ相対式**。入力は「画面のどっちへ行きたいか」を表し、
キャラクターは移動方向へ自動で振り向く(振り向き速度: 指数減衰係数10)。

| 入力 | 動作 |
| --- | --- |
| W / ↑ | 画面の奥へ移動 |
| S / ↓ | 画面の手前へ移動(振り返ってカメラ側へ走る) |
| A / ← | 画面の左へ移動 |
| D / → | 画面の右へ移動 |
| マウス/タッチのドラッグ | カメラをプレイヤーの周りに周回(感度 0.005 rad/px) |
| ホイール | ズーム(距離3.5〜18、指数スケール) |
| 仮想スティック(タッチ端末) | 移動(可動半径40px、デッドゾーン0.2) |

- 移動速度: 6 単位/秒
- キーボードとスティックの入力は加算合成され、各軸±1にクランプされる
- ウィンドウの `blur` で全入力を解除する(安全のため)
- カメラの視線方向はキャラクターの向きから独立しており、キャラが曲がっても画面は回らない
- カメラ見下ろし角: -0.25〜1.35 rad(真上直前まで)。惑星内部には入らない(中心からの距離下限 = 半径+1.5)

## 4. ワールド仕様

- 惑星半径: 25(ワールド原点が惑星の中心)
- 昼夜サイクル: 太陽が240秒で一周(空のグラデーションも連動)
- 配置はシード固定乱数(シード値 20260714)により毎回同じ

### 4.3 配置ルール(重なり防止)

- 全配置物は「開始地点」「湖」「丘」「集落」「畑」「牧場」「おえんちゃんの家」を避ける
- 道路からの最低距離: 村の家1.8 / 街道の家1.6 / 一軒家2.5 / 岩1.4 / 木1.3 / キノコ1.0
- 畑・牧場・緑屋根の家は、道に近い場合は自動で法線方向へ押し出される
- 道が湖を横切る場合は、湖の外周へ膨らむ経由点を自動挿入して迂回する
- 道路タイルは既存タイルと0.3未満の距離には敷かない(交差点の過密防止)

## 5. キャラクター仕様

### 5.1 プレイヤー(おえんちゃん)

- 約2.5頭身の積み木人形。当たり判定半径0.35
- パーツ: 黄緑ボブヘア(土台球+前髪の房5本+横髪+後頭部)、猫耳(ピンクの内耳つき)、
  Canvas生成の顔テクスチャ(琥珀色の目・細い茶色の丸眼鏡・口・頬)、
  緑の着物(本体)+前開きのオレンジ羽織+黒い縁取り+羽織紐+白帯、
  茶色のリュック+肩ベルト、白い先端の黄緑の尻尾、黒ニーソックス+濃茶の履物
- 歩行時は腕・脚を振り、上下に弾む。旋回時は体が内側へ傾く。静止時は呼吸

### 5.2 村人NPC

- おえんちゃんと同じ頭身の積み木人形。個体差:
  - 服5色 × 帽子3種(とんがりフード/丸帽子/ボブ髪)×6色 × 前掛け有無(50%)
  - 体型4種: ふつう / 背が低い / ノッポ / ぽっちゃり(+背丈±8%)
- AI: 待機1.5〜5秒(ときどきキョロキョロ)⇔ 家の周り半径約1.5を散歩(速度1.2〜1.95)
- 障害物に阻まれ進めないと判断したら(0.7秒ごとの進捗チェックで期待の40%未満)別の行き先へ
- プレイヤーとNPCは互いにすり抜けない(動的コライダー半径0.4)

## 6. パフォーマンス仕様

- NPC・動物・虫は、プレイヤーから約60°以上離れると描画とAIを停止する
  (dot判定: NPC 0.5 / 動物・虫 0.45)。虫の可視判定は現在時刻の経路位置で行う
- 草・道路タイルはInstancedMesh(各1ドローコール)
- トゥーンマテリアルは色+自己発光をキーに全体でキャッシュ共有
  (両面描画は専用キャッシュ。共有マテリアルのプロパティ書き換えは禁止)
- ジオメトリはモジュール単位で共有(NPC20体は同一ジオメトリセット)
- 実測: 60Hz環境でフレーム落ちなし(120Hz環境で120FPS動作の実績)

## 7. 意図的に実装していないもの

- ジャンプ・ダッシュ、物理エンジン、地形の高低差(球面移動の安定性を最優先)
- 外部3Dモデル・画像テクスチャ・GLB/glTF
- 音楽・効果音
- ゴール・スコア・敵・アイテム・会話・クエスト等のゲームシステム
