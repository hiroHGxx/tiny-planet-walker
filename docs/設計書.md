# Tiny Planet Walker 設計書

作成日: 2026-07-16(ソースコードからのリバースエンジニアリングにより作成)
対象バージョン: 0.1.0

この文書は「なぜこの作りなのか」「どこを触ればどう変わるか」を残すための設計メモ。
機能の一覧・数値は `docs/仕様書.md` を参照。

## 1. モジュール構成

```
src/
├── main.ts        入口。レンダラー初期化(WebGL失敗時の案内)、入力(キー+仮想スティック)、メインループ
├── palette.ts     カラーパレット定数・トゥーンマテリアルのキャッシュ・草色ノイズ・PLANET_RADIUS
├── player.ts      プレイヤーの見た目(おえんちゃん)とカメラ相対の球面移動 ★最重要
├── camera.ts      視線方向を平行移動で保つ三人称オービットカメラ
├── collision.ts   球面上の円形コライダー(登録・反復押し出し)
├── world.ts       ワールド生成の統括。配置ルール・毎フレーム更新の合成
├── layout.ts      星のレイアウト計画(村・湖・丘・道・畑・群生地の設計図と回避ヘルパー)
├── town.ts        町の部品(道・湖・丘・家・集落・柵・畑・おえんちゃんの家)
├── npc.ts         村人NPC(外見バリエーション+ルールベースAI)
├── dialogue.ts    村人のつぶやき(HTML吹き出しをスクリーンへ投影)
├── journal.ts     薬草図鑑(発見判定・パネルUI・localStorage保存)
├── audio.ts       環境音とBGMの鳴らし方(メロディの中身は content/melodies.ts)
├── animals.ts     動物(鳥・うさぎ・ひつじ)と虫(ハチ・トンボ)
├── flora.ts       薬草・木・岩・キノコ・小物のファクトリ(共有ジオメトリ)
├── shop.ts        薬屋の建物(おえんちゃんの家として使用)
├── grass.ts       InstancedMeshの草原+風の頂点シェーダー+間引きゾーン
├── ambient.ts     雲・蝶・流れ星
└── effects.ts     ポストプロセス(深度エッジ輪郭線・ブルーム・ビネット)
```

依存の向き: `main → world → (layout, town, npc, animals, flora, shop, grass, ambient)`、
全員が `palette / collision / placement` を利用する。循環依存はない
(player/camera が参照する `PLANET_RADIUS` は world が palette から再エクスポート。
レイアウトの公開定数も world が layout から再エクスポートし、features/テストの
参照先は world のままでよい)。
`main → (dialogue, journal, audio)` は演出レイヤーで、worldの戻り値
`{ update, npcs, herbSightings }` から参照だけを受け取る
(world → journal は型のみの依存)。

データの台帳は `content/` に集約する:
`herbs.ts`(薬草の名前・説明・挿絵SVG・工場。図鑑・ポーチ・群生地が参照)/
`planets.ts`(星ごとのしきい値・薬草・テーマ・レイアウト上書き)/
`melodies.ts`(BGMの拍と音符)/ `items・recipes・npcs・quests/`。
薬草を増やす手順: flora.ts に工場 → content/herbs.ts に1エントリ →
content/planets.ts の herbs に id を足す(これだけで図鑑・ポーチ・依頼・群生地に載る)。

## 2. 核心設計: 球面移動

**「上方向 = 球面法線」の一点に集約する。** 惑星中心が原点なので、
位置ベクトルの正規化がそのまま「その地点の上」になる。

1. 移動は接平面上で行い、毎フレーム `position.normalize().multiplyScalar(25)` で半径へ戻す
   → 何時間歩いても浮き・めり込みが起きない
2. 姿勢は「現在の上 → 新しい上」への最小回転(`Quaternion.setFromUnitVectors`)を掛けるだけ
   → 前方向の連続性が保たれ、極を通っても反転しない。オイラー角・lookAtの世界上方向は使わない
3. 毎フレーム quaternion.normalize() で数値誤差の蓄積を防ぐ
4. すべて deltaTime 比例(フレームレート非依存)。一時Vector3/Quaternionはモジュール内で使い回す(GC対策)

### カメラ相対入力(player.ts)

- カメラの視線方向を接平面へ投影 = 「画面の奥」。法線との外積 = 「画面の右」
- 入力ベクトル(x, z)をこの基底で移動方向に変換し、即座にその方向へ移動
- キャラの向きは移動方向への符号つき角度(`atan2(cross·up, dot)`)を指数減衰で追従

### カメラ(camera.ts)

- **視線方向(viewForward)はキャラの向きから独立**。これがカメラ相対移動の前提
  (依存させると、左入力→キャラ回転→カメラ回転→入力の意味が変わり、ぐるぐる回る)
- プレイヤーが球面上を動いた分は「前の法線→今の法線」の最小回転で視線方向を運ぶ(平行移動)
- ドラッグは法線軸の回転として視線方向へ適用。位置・注視点・上方向は指数減衰補間
- メインループの順序: `player.update(前フレームのviewForward)` → `camera.update` → `world.update`

## 3. 衝突(collision.ts)

- コライダー = 「球面上の中心方向(単位ベクトル)+ 表面に沿った半径」の円
- 判定は内積1つ。押し出しは「障害物中心×位置」を軸にした回転なので**半径が変わらない**
- 斜めに当たると自然に横へスライドする
- 押し出し先で別の円に入り直すことがあるため、最大3回反復
- `addCollider` はオブジェクトを返し、`direction` を書き換えれば動く障害物になる(NPCが使用)
- `resolveCollisions(pos, r, ignore)` の ignore で自己衝突を除外(NPC用)

## 4. 配置システム(world.ts / town.ts / placement.ts)

- `placeOnPlanet(obj, direction, {sink, yaw})`: 原点が足元・+Yが上に作ったオブジェクトを
  球面法線に合わせて立たせる共通処理。全配置がこれを通る
- 乱数はシード固定(20260714)の線形合同法 → 毎回同じ星になる。
  **注意: 配置コードの途中に乱数消費を挿入すると、それ以降の配置がすべて変わる**

### レイアウトの定数(layout.ts モジュールスコープ)

`VILLAGE_CENTERS / LAKES / HILLS / FOREST_CENTERS / OEN_HOME / FARM_FIELDS / PASTURES /
FLOWER_FIELDS / SHOP_FRONT / OEN_JUNCTION / BASE_ROAD_ARCS → ROAD_ARCS` を
layout.ts のモジュール定数として持ち、配置関数と回避判定が同じ値を参照する
(公開分は world.ts から再エクスポート)。星ごとの回転(LAYOUT_ROTATION)と
湖の上書き(content/planets.ts の layout.lakes)もここで適用される。

### 重なり防止の3つの仕組み

1. `isInsideTownFeature(dir, margin)`: 湖・丘・集落・畑・牧場・おえんちゃんの家との重なり判定
2. `distanceToRoads(dir)` / `nearestRoadInfo(dir)`: 大円法線とのasinで道までの表面距離
   (弧の範囲外は無視)。家・岩・木・キノコの配置チェックに使う
3. `pushedOffRoads(dir, clearance)`: 道に近すぎる固定レイアウト(畑・牧場・緑屋根の家)を
   法線方向へ反復シフトして自動で離す

### 道路(town.ts addRoads)

- 弧(大円)ごとにタイルを3列(中央のみ半歩ずらし)で敷く。全タイルInstancedMesh 1つ
- レンガ3色+明度ゆらぎ+1枚ごとの角度ジッターで手作り感
- 既存タイルと0.3未満の位置はスキップ(交差点の過密防止。同一道路の最小間隔0.32は温存)
- `routeAvoidingLakes`: 弧が湖のクリアランス円を横切る場合、最接近点を円の外へ
  押し出した経由点を挟む。分割後の弧も対象に、変化がなくなるまで反復(最大6パス)
- 3枚に1枚のタイル方向を返し、草の間引きゾーンとして使う

### 草(grass.ts)

- `GrassAvoidZone { direction, surfaceRadius, keepRatio }` の配列を受け取り、
  ゾーン内は keepRatio の確率でしか生やさない(道0.1 / 薬屋跡0 / 湖0 / 畑0 / 群生地0.35 など)
- 風は `onBeforeCompile` で頂点シェーダーに注入。`customProgramCacheKey` で
  他のトゥーンマテリアルとプログラムを取り違えないようにしている

## 5. AI設計

### NPC(npc.ts)

状態機械: `idle`(1.5〜5秒。1.2〜3.7秒ごとにランダムな接線方向を向く)⇔
`walk`(家の周りWANDER_RADIUS=0.06radのランダム地点へ大円歩行。速度は散歩ごとにゆらす)

- スタック検知: 0.7秒ごとに実移動量を確認し、期待の40%未満なら休憩して別の行き先へ
  (障害物に向かって歩き続けない)
- 向きは `Matrix4.lookAt(位置, 位置+接線, 法線)` + slerp
- ワールド更新は経過時間しか受け取らないため、`lastTime` との差分で deltaTime を作る
- 距離カリング: `dot(自位置, プレイヤー方向) < 0.5` で `mesh.visible=false` + 更新スキップ

### 動物(animals.ts)

NPCの簡易版(キョロキョロなし、コライダーなし=resolveCollisionsの片側判定のみ)。
種ごとの `AnimalSpec { speed, hopHeight, gaitFrequency, wanderRadius }` で歩様が変わる。
家畜ひつじは wanderRadius を柵の半径より小さくして外へ出さない。

### 虫(animals.ts / ambient.ts)

状態を持たず「時間の関数で決まる経路」を飛ぶ(ハチ=基準点まわりの円、
トンボ・蝶=緯度経度のゆらぎ)。**可視判定は必ず現在時刻の経路位置で行う**
(描画済みの古い位置で判定すると、再表示されない・ワープするバグになる)。

## 6. 見た目の設計

- **トゥーンマテリアルのキャッシュ**(palette.ts): 色+自己発光がキー。
  戻り値は共有物なのでプロパティを書き換えないこと。両面描画(羽)は
  `doubleSidedToonMaterial`(別キャッシュ)を使う
- **フラットシェーディング**: MeshToonMaterialはflatShading非対応のため、
  `flatGeometry()` = toNonIndexed + computeVertexNormals でローポリの面を出す
- **顔のCanvasテクスチャ**(player.ts / npc.ts): SphereGeometryのUVは
  「正面(-Z)= u 0.75 / 頭頂 = キャンバス上端」。眼鏡もテクスチャに描く
  (立体パーツより細くできる)。`getContext('2d')` がnullの環境(テスト)では無地で代用
- **ポストプロセス**(effects.ts): 通常描画 → 深度エッジ検出の輪郭線
  (Points・半透明・userData.noOutline は深度パスから除外)→ UnrealBloom →
  ビネット+彩度 → OutputPass
- **空**(world.ts): 大きな球の内側にシェーダー。uniformがSUN_DIRECTIONを参照共有して
  いるため、太陽の周回で空の明るい側も自動で動く
- **影**: `enableShadows()` が不透明トゥーンメッシュへ一括で castShadow を付与。
  惑星本体・草・薄いタイルは `userData.noCastShadow` で除外

## 7. メインループのデータフロー

```
timer.update → deltaTime(上限0.1でワープ防止)
→ player.update(deltaTime, 入力, カメラ視線方向)   … 移動・振り向き・衝突・姿勢
→ followCamera.update(deltaTime, player)           … 視線の平行移動・補間
→ world.update(経過時間, プレイヤー位置)            … 太陽・煙・粒・草・NPC・動物・虫・雲・蝶・流れ星
→ speechBubbles / journal / ambientAudio の update  … つぶやき・図鑑・環境音(昼夜はgetSunElevation)
→ effects.render()
```

`window.__player`(DEVビルドのみ)からプレイヤーを参照でき、動作検証に使う。

## 7.5 つぶやき・図鑑・環境音の設計(2026-07-17追加)

- **実行時演出はシード乱数を消費しない。** セリフ選びやさえずりの間隔は
  `Math.random()` を使う。配置用の `rand` をうっかり渡すと星のレイアウトが変わる
- **dialogue.ts**: 吹き出しはHTML(#speech-layer)。`Npc.getBubbleAnchor()` の
  ワールド座標を毎フレーム `Vector3.project(camera)` でスクリーンへ投影する。
  カメラの後ろ(projected.z > 1)では非表示にする
- **journal.ts**: 薬草のID・名前・説明・挿絵・工場は content/herbs.ts の台帳が持ち、
  journal は従来の形(HERB_SPECIES / HERB_ICONS)で派生・再エクスポートする。発見判定は
  未発見種の株だけを残した配列への内積チェックで、見つけた種はまとめて除外する
- **audio.ts**: AudioContextは最初のユーザー操作で作る(自動再生制限)。
  風のノイズバッファは足音と共有する。localStorageが使えない環境でも動くよう
  すべてtry/catchで包む
- **world.ts の戻り値**は update 関数単体から `{ update, npcs, herbSightings }` に
  変更した(2026-07-17)。演出レイヤーが参照を受け取るための変更で、
  update の中身と呼び出し順は変えていない

## 8. テスト設計(Vitest + happy-dom)

| テスト | 保証すること |
| --- | --- |
| player.test.ts | 一周以上(2,200フレーム)歩いても半径25、極通過後も姿勢ノルム1。横移動でも同様 |
| collision.test.ts | 円からの押し出し(半径不変)、密集配置での反復解決 |
| town.test.ts | moveTowardの角度精度、全道路弧が湖(砂+道幅)へ入り込まないこと |

テストは実行時にDOMが必要なため happy-dom を使用。顔テクスチャは2Dコンテキスト
なしでも動くようガード済み。`world.ts` は `ROAD_ARCS` / `LAKES` をテスト用にエクスポートする。

## 9. 既知の制約・変更時の注意

- **地形の高低差は非対応**(丘は登れない景観)。対応する場合は player の半径正規化、
  NPC・動物の移動、草の配置まで波及する大改造になる
- 配置コードに乱数消費を追加すると、それ以降の星のレイアウトがすべて変わる
- 共有トゥーンマテリアルのプロパティ書き換え禁止(side等は専用ヘルパーを使う)
- レイアウト定数(村・湖など)を動かしたら `npm test`(道と湖の重なり検証)を実行する
- ブラウザ自動検証の注意点は `~/.claude` のプロジェクトメモリ
  (tiny-planet-walker-verify-workflow)を参照
